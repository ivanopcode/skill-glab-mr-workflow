# MR Review Checklist

Use this order for merge request reviews:

1. Confirm the exact target MR URL, repo, and head SHA.
2. Check pipeline status before commenting on code style.
3. If the pipeline failed, extract the real failure cause from the failed job trace.
4. Read unresolved discussions before reading the full diff.
5. Read the raw diff for the changed files that matter.
6. Report findings first, ordered by severity.
7. Approve or merge only when the user explicitly asks for that write action.
